Sensus AMR Water Meter Reading

Standard Operating Procedure
Last Updated: April 2026  |  Original Author: Jeff Lannigan (March 2021)  |  Updated by: Kent Overby (April 2025), John Slagboom (April 2026)
What Is This?

WSU has about 165 water meters buried in pits and vaults across the Pullman campus. Every month we read all of them using a radio system called Sensus AMR (Automated Meter Reading). A person drives around campus with a tablet and a small blue radio box. The radio box talks to transmitters on the meters and collects the readings automatically. Then we bring the data back to the office, pull it into a PC, and export it as a file.

This document tells you exactly how to do every step of that process.

Equipment You Need

You will be working with three pieces of equipment:

1. Android Tablet

  • Has the FieldLogic app installed (icon looks like three green bars)
  • This is what you carry in the field
  • It talks to the Command Link via Bluetooth
  • Screen lock PIN: 123456

2. One Command Link (CL)

  • Small blue box
  • It communicates with the tablet via Bluetooth and with the water meters via radio frequency (RF)
  • Power button is hidden under a white square on the side
  • When communicating, a blue "LINK" lamp lights up
  • Bluetooth pairing password: 0000

3. Office PC (Slagboom's PC)

  • Has two programs installed:
    1. AutoRead — imports routes to and exports data from the tablet
    2. FieldLogic Hub — sets up meter types and system configuration
      Login: Admin  /  Password: Sensus123!

\\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\

Phase 1: Prepare the Route (Office — Before Going to the Field)
Step 1 — Update the Route File (only if meters were added/removed/changed)

The master route file is an Excel spreadsheet. We keep it in Excel because Excel preserves leading zeros in building numbers (like 0358C). If you open the CSV version in Excel, it destroys those zeros.

Master Excel file:

\\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Routes\import_amr.rte
Yes, the .rte file IS a CSV file — AutoRead just needs that extension.

If you need to add, remove, or change a meter:

  1. Open the Excel version of the route (or edit the .rte in a text editor like Notepad — NOT in Excel, which drops leading zeros).
  2. Make your changes (see Route File Maintenance section below).
  3. Save.
  4. IMMEDIATELY make a backup copy in:
    \\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Routes\Do not screw this file up\
    Name it: import_amr - Master Backup M-D-YY.rte

If no meters changed since last month, skip this step entirely.

Step 2 — Connect the Tablet to the PC via WiFi

The tablet connects to the PC wirelessly, NOT by USB cable.

  1. On the office PC, turn on the Mobile Hotspot:
    • Open Windows Settings > Network & Internet > Mobile hotspot
    • Turn it ON
    • Note the network name and password shown
  2. On the Android tablet:
    • Open Settings > Wi-Fi
    • Connect to the PC's hotspot network using the password above
  3. Verify the connection:
    • Open AutoRead on the PC:
      1. Open File Explorer
      2. Navigate to C:\AutoRead\
      3. Double-click AutoRead.exe
      (There is no desktop shortcut — you must open it from this folder.)
    • In the left-hand menu, click "Ethernet/Wi-Fi (FieldLogic)"
    • The tablet should appear in the list
    • If it does NOT appear:
      1. Make sure the PC hotspot is ON
      2. Make sure the tablet is connected to that hotspot (not some other Wi-Fi network)
      3. On the tablet, close the FieldLogic app completely and reopen it
      4. Try again
Step 3 — Clear, Import, and Load the Route

Open AutoRead on the PC (C:\AutoRead\AutoRead.exe — see Step 2 above).

  1. In the left-hand tree menu, navigate to:
    Route Processing > Handheld > HHD003 CSV
  2. CLEAR old readings:
    • From the Function dropdown at the top, select "Clear"
    • Click Execute
    • This wipes last month's readings from the system
    IMPORTANT: Before you do this, make sure last month's data has already been exported and saved! Once you clear, it's gone.
  3. IMPORT the route:
    • From the Function dropdown, select "Import"
    • Click Execute
    • This loads the route file from:
      \\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Routes\import_amr.rte
    • AutoRead now has all ~165 meter accounts with building info, meter IDs, and MXU (radio) IDs, but no current readings yet
  4. LOAD the route to the tablet:
    • From the Function dropdown, select "Load Data"
    • Click Execute
    • This copies the route from AutoRead to the tablet over WiFi
    • CHECK: Look at the meter count on the tablet screen. It should be around 165. If it's way off, something went wrong with the import.
    • If there are import errors, you can drill into them in AutoRead to see which meters failed and why.
Phase 2: Collect Readings in the Field
Step 4 — Read the Meters
  1. Make sure the Command Link is charged and powered ON. (Press the hidden button under the white square on the side.)
  2. On the tablet, open the FieldLogic Tools app (three green bars icon).
  3. Open the route you just loaded.
  4. Tap "Start Read."
  5. Make sure the Command Link shows a blue "LINK" light — this means it's connected to the tablet via Bluetooth.
  6. Drive around campus. The tablet will automatically pick up readings from meters via the Command Link as you drive within range.
    • By default, meters drop off the screen once they're read.
    • To see ALL meters (read and unread), tap the funnel icon in the upper-right corner of the screen and change the filter to "None."
  7. Most meters will read from inside the car. Known exceptions:
    • Vogel Plant — may need to get closer
    • ITB — sometimes doesn't read from the car
  8. If a meter won't read via radio, you can enter the reading manually on the tablet. Go to the meter pit/vault, read the register, and type it in.
  9. When you're done, check the tablet to make sure all meters have readings. Any still showing as unread need attention.
Phase 3: Export the Data (Office — After Returning from the Field)
Step 5 — Connect the Tablet and Pull Data into AutoRead
  1. Connect the tablet to the PC via WiFi hotspot (same as Step 2).
  2. Open AutoRead on the PC (C:\AutoRead\AutoRead.exe — see Step 2).
  3. In the left-hand tree menu, navigate to:
    Route Processing > Handheld > HHD003 CSV
  4. Verify the tablet appears under "Ethernet/Wi-Fi (FieldLogic)" in the left menu. If it doesn't, troubleshoot the WiFi connection (see Step 2 above).
  5. From the Function dropdown, select "Read Data" and click Execute.
    • This pulls the readings from the tablet into AutoRead.
    • A window may ask: Overwrite, Append, or Skip? SELECT "OVERWRITE."
  6. AutoRead will automatically prompt you to save two PDF reports:
    1. MASTER ROUTE REPORT
      • This shows every meter with its current reading
      • Save it with a dated name to:
        \\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Master Route Report 7-6-23\Master Route Report M-D-YY.pdf
        Example: Master Route Report 4-3-26.pdf
    2. ROUTE STATISTICAL SUMMARY REPORT
      • This shows summary statistics (how many meters read, skipped, etc.)
      • Save to the same folder:
        \\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Master Route Report 7-6-23\Route Statistical Summary Report M-D-YY.pdf
      • Unless you're troubleshooting issues, you can overwrite the previous version of this report.
    Both PDFs will open automatically so you can eyeball the data.
Step 6 — Export Readings to a File
  1. From the Function dropdown in AutoRead, select "Export" and click Execute.
  2. A window will ask: Overwrite, Append, or Skip? SELECT "OVERWRITE."
  3. AutoRead saves the file as:
    \\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Output\latestread_amr.dat
    Yes, it's a .dat file. The data inside is plain comma-separated text — the extension is just what AutoRead uses by default.
Step 7 — Convert the Exported File from .DAT to .CSV
  1. Open File Explorer and navigate to:
    \\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Output\
  2. Find the file called "latestread_amr.dat"
  3. Right-click the file and select "Open with" > Notepad.
  4. Glance at the data — make sure you see building numbers, readings, and dates. If it's empty or garbled, something went wrong with the export.
  5. In Notepad, click File > Save As...
  6. In the Save As dialog:
    1. In the "Save as type" dropdown, select "All Files (*.*)"
    2. Change the file name to: AMRoutputYYYYMMDD.csv
      where YYYYMMDD is today's date.
      Example: AMRoutput20260403.csv
    3. Make sure you are still in the same Output folder:
      \\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Output\
    4. Click Save.
  7. Delete the original latestread_amr.dat file (you now have the .csv version).
Step 8 — Archive Last Month's File
  1. In the same Output folder, you should see last month's CSV file (e.g., AMRoutput20260303.csv).
  2. Move (cut and paste) last month's file into:
    \\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Output\Old Outputs\
  3. When you're done, the Output folder should have ONLY:
    • Your new CSV file (e.g., AMRoutput20260403.csv)
    • The README text files (leave those alone)
RULE: Only ONE month's data file should be in the Output root folder at a time. Everything else goes in Old Outputs.
Phase 4: New Meter Onboarding (Only When Installing a New Meter)
Step 9 — Activate a New Meter in the Field

This is only done when a brand new meter is physically installed.

  1. BEFORE going into the building, start the tablet outside and wait for it to get a GPS fix. FieldLogic won't let you add a meter without recent GPS coordinates.
  2. Place the Command Link in physical contact with the meter's radio transmitter (MXU). Sometimes it's sensitive to position — if it doesn't connect, try moving the CL around on the transmitter.
  3. On the tablet in FieldLogic, press "Connect."
  4. The app will walk through the connection process. What happens next depends on whether this meter type is already configured:
    • If the meter type IS already set up:
      1. You may get a prompt — select "Activate"
      2. The app may ask you to confirm the physical installation is complete — if it is, tap OK
      3. You should see "Activation Success" — tap OK, then "Disconnect"
    • If the meter type is NOT set up:
      You need to configure it in FieldLogic Hub first (see the AutoRead Software Configuration section below).
  5. After activation, add the meter to the route file:
    • Edit the route file (see Route File Maintenance below)
    • Add a new row with the meter's Account ID, Building #, MXU ID, Meter ID, and other details
    • The new meter will be included in the next month's reading
IMPORTANT — METER IDS:
  • Every meter needs a UNIQUE Meter ID number
  • Every radio transmitter needs a UNIQUE MXU ID (radio ID) number
  • Sensus meters come with their own factory-assigned IDs
  • Neptune meters may NOT have unique IDs — this has been a known issue. Double-check that IDs don't conflict with existing meters.
Route File Maintenance
The Route File Explained

The route is a CSV file with 27 comma-separated fields per line. The first line is a header row. Each subsequent line is one meter.

File location:

\\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Routes\import_amr.rte

Key Fields

ColumnFieldDescription
1Account IDUnique number (e.g., 200001)
6Building #WSU building number (e.g., 0358C)
7Service AddressBuilding name (e.g., CHEM STO)
8Location DescWhere the meter is (e.g., Ext Vlt, Mech Rm)
9Meter NumberWSU meter ID (e.g., 0358CDW_001)
10Service TypeAlways W (Water)
17MXU IDRadio transmitter serial number
18Meter IDMeter register serial number
19Meter TypeB = Sensus, P = Neptune, M = Manual
20MXU TypeC = Commercial MXU, B = Battery MXU
25SystemDW = Domestic Water, IR = Irrigation
26Multiplier1, 100, or 1000 (converts raw reading to gallons)

Adding a New Meter

  1. Open import_amr.rte in Notepad (NOT Excel — Excel drops leading zeros).
  2. Go to the last line of data.
  3. Add a new line with the next Account ID (e.g., if last is 200165, use 200166).
  4. Fill in all the fields following the format above.
  5. Leave columns 12–13 (readings) and 21–22 (date/time) BLANK.
  6. Save the file.
  7. Make a backup copy in:
    \\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Routes\Do not screw this file up\
    Name it: import_amr - Master Backup M-D-YY.rte

Removing a Meter

  1. Open import_amr.rte in Notepad.
  2. Find the line for the meter (search for the Account ID or Building #).
  3. Delete the entire line.
  4. Save and make a backup (same folder as above).

Changing Meter Hardware (radio or register replaced)

  1. Open import_amr.rte in Notepad.
  2. Find the meter's line.
  3. Update the MXU ID (column 17) and/or Meter ID (column 18).
  4. Update Meter Type (column 19) if the brand changed.
  5. Save and make a backup.
ALSO UPDATE THE MASTER SPREADSHEET: Whenever you change the route file, also update:
\\fais007\ENERGY GROUP\Energy Services Admin\Energy Files\Meters\SENSUS\Sensus Exhibit A Spread Sheet Master.xlsx

Troubleshooting
Tablet doesn't appear in AutoRead
  1. Confirm PC Mobile Hotspot is ON.
  2. Confirm tablet is connected to that hotspot (not campus WiFi or other).
  3. Close FieldLogic app on tablet and reopen it.
  4. Restart AutoRead on the PC.
Command Link won't pair
Bluetooth password is 0000. Press the hidden power button (under white square on the side). Blue LINK lamp should glow.
Meter won't read via radio
  1. Drive closer to the meter.
  2. Check that the MXU is powered and the antenna isn't damaged.
  3. Enter the reading manually on tablet.
Known trouble spots: Vogel Plant, ITB.
Import shows errors in AutoRead
Drill into the error list in AutoRead. Common cause: duplicate MXU ID or Meter ID in the route file. Each must be unique.
Export produces latestread_amr.dat
This is normal. Convert to .csv using the Notepad Save As method (see Step 7).
Reading is zero or unchanged
Possible stuck register. Compare against previous month. Physical inspection may be needed.
Leading zeros missing after edit
You opened the CSV in Excel. Excel destroys leading zeros. Always edit the route file in Notepad, never in Excel.
Can't add a meter (GPS error)
Start the tablet outside and wait for a GPS fix before going into the building.
FieldLogic Hub won't open
Login: Admin / Password: Sensus123!
Make sure FieldLogic Hub is installed (installer on the shared drive, see AutoRead Software Configuration above).
Neptune meter ID conflict
Neptune meters may share IDs with other meters. Assign a new unique ID before adding to the route.
